# Forge UX Mockups

These mockups are visual direction references for Forge. They are not pixel-final designs.

The goal is to preserve the product feel before implementation starts:

- Fast, beautiful, low-ceremony execution console.
- Calm, dense, developer-native surfaces.
- Timeline and ledger patterns over boards.
- Proof-first views for attempts, checkpoints, blockers, and artifacts.
- No Jira-style backlog grooming, sprint machinery, or field sprawl.

## Design Notes

Use these images to guide information architecture and tone:

- The TUI should feel like a keyboard-first operator console.
- The web UI should feel like a polished shared inspection surface.
- Proposed work should feel like a breezy review inbox, not backlog grooming.
- The execution ledger should feel calm and operational, not like a noisy observability dashboard.
