change recipes

Common changes, correctly scoped.

New lifecycle rule
internal/services first, then runtime/adapters; extend service and contract tests. Do not make web-only state rules.
New query or field
Write a forward migration in sql/migrations, add sql/queries, regenerate sqlc code, then exercise it through services and integration tests.
New human route
Implement internal/web behavior and explicitly register it in internal/api/router.go; add a router-level test so a handler-only route cannot silently 404.
New agent operation
Define its stable shape in internal/contracts, then bind compatible CLI, REST, and MCP adapters to shared runtime behavior.
