local language

The terms that make Forge legible.

Definitions reflect the current repository and call out boundaries where a word denotes a foundation rather than a finished platform capability.

ledger and lifecycle

Ticket

A scoped work unit with type, priority, lifecycle state, acceptance criteria, verification commands, and event history.

Attempt

An agent’s leased ticket execution. It can heartbeat, checkpoint, attach proof, and reach a terminal state.

Fence / lease

Concurrency control that prevents stale or competing agents from writing after ownership expires or changes.

Idempotency replay

Repeating a claim with the same stable key returns the original result instead of claiming additional work.

surfaces and integrations

MCP

Model Context Protocol. Forge exposes a stdio MCP server for contract-defined agent operations.

Huma

The Go API framework that registers the typed OpenAPI/REST surface under /api/v1.

sqlc

Generates Go query code and models from SQL; it bridges migrations/queries and internal/db.

HTMX

Used by the server-rendered human web UI for progressive interactions without a React frontend.

operations

River

PostgreSQL-backed job runner used for maintenance and webhook delivery workers.

Artifact

Ledger metadata plus bytes held in local storage or an S3-compatible store; evidence, logs, patches, and handoffs are artifact roles/types.

S3-compatible

An object-storage API compatible with Amazon S3. Forge also supports a local artifact root.

Readiness

/readyz means the server can reach PostgreSQL; /livez only means the process is serving.

product boundaries

Proposed work

Agent-discovered follow-up triaged before it becomes active queue work, preventing automatic Jira-like backlog growth.

Workspace membership

A current foundation for scoped actors and roles—not full RBAC yet.

Trusted single-tenant

The v0.1 model: a shared admin-token boundary and minimal authorization complexity, with broader RBAC deferred.

Dogfood pilot

The operational trial that validates real use and produces the next production go/no-go verdict.
