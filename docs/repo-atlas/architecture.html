current architecture

One runtime, many entry points.

CLI, HTTP API, MCP stdio server, TUI, and server-rendered web handler converge on Runtime and services. PostgreSQL is the authoritative ledger; object storage holds proof bytes.

From agent claim to verifiable outcome.

Invariant: human UI code does not invent transitions; it invokes runtime-backed actions that validate and write ledger events.

Ticket lifecycle

backlog → todo → in_progress → needs_review → done

Alternative states are blocked, failed, and archived; explicit ready/reopen/unblock/review actions write events and apply policy.

Attempt lifecycle

running → succeeded | failed | blocked | expired | cancelled

Claims use locking and idempotency replay. Heartbeats extend leases; checkpoints preserve context; terminal writes can atomically register proof.
